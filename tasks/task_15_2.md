# Задача 15.2: Stream telemetry pass-through (metrics populated)

## Связь с юзер-кейсами
- UC-02: Observe a live public session
- UC-08: Subject breath telemetry

## Цель задачи
Сохранить телеметрию при публикации событий стрима, чтобы метрики в UI не были пустыми.

## Описание изменений

### Изменения в существующих файлах

#### Файл: `apps/server/src/routes/stream.ts`
- Передавать `telemetry` при публикации события в StreamService.
- Логировать пустые события без содержания, но не отбрасывать телеметрию.

## Тест-кейсы

### Интеграционные
1. **TC-INT-15.2-01:** Публикация Subject события с `telemetry` сохраняет поле в WebSocket payload.

## Критерии приёмки
- [ ] Телеметрия присутствует в событиях WebSocket и реплее.
- [ ] Диагностические метрики в UI заполняются при наличии telemetry.
