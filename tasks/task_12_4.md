# Задача 12.4: Playwright + deploy debug loop for Subject replies

## Связь с юзер-кейсами
- UC-02: Observe a live public session
- UC-09: Configure language mode and theme

## Цель задачи
Подтвердить, что Subject ответы отображаются корректно после изменений, выполнить e2e и повторный деплой с проверкой логов.

## Описание изменений

### Изменения в существующих файлах

#### Файлы: `tests/e2e/*.spec.ts`
- Добавить проверки переключателей языка/темы.
- Добавить проверку, что Subject reply не пустой при наличии события.

#### Файлы: `docs/runbooks/ops.md`, `docs/runbooks/deploy.md`
- Обновить лог-запросы для новых событий (`subject_request`, `subject_empty_response`).
- Добавить упоминание языковых/тематических переменных окружения (если добавлены).

## Тест-кейсы

### End-to-end тесты
1. **TC-E2E-12.4-01:** Переключение языка меняет ключевые метки UI на RU.
2. **TC-E2E-12.4-02:** Переключение темы изменяет `data-theme` на `dark`.
3. **TC-E2E-12.4-03:** Subject reply отображается непустым.

### Релизный цикл
- Запустить local Playwright (headed) + визуальная проверка.
- Деплой через `infra/gcp/deploy.sh`.
- Запустить Playwright против проды + визуальная проверка.
- Проверить Cloud Logging на события Subject пайплайна.

## Критерии приёмки
- [ ] Playwright локально и в проде проходит.
- [ ] В логах есть события Subject пайплайна без повторяющихся пустых ответов.
- [ ] Переключатели языка/темы работают в проде.
