# Задача 29.2: Repo governance + env example + runbook sanitization

## Связь с юзер-кейсами
- UC-16: Open-source repository onboarding
- UC-17: Public deployment readiness

## Цель задачи
Добавить governance-документы для публичного репозитория, пример env-файла, и санитизировать runbooks для внешней аудитории.

## Описание изменений

### Новые файлы
- `.env.example` — список обязательных переменных окружения с безопасными плейсхолдерами.
- `CONTRIBUTING.md` — инструкция по внесению изменений и запуску тестов.
- `SECURITY.md` — политика уязвимостей и канал для приватных отчётов.
- `CODE_OF_CONDUCT.md` — выбранная политика поведения.
- `LICENSE` — выбранная лицензия.

### Изменения в существующих файлах

#### Файл: `docs/runbooks/deploy.md`
- Заменить внутренние идентификаторы (project ID, service URL, instance IDs, VPC connector) на плейсхолдеры.
- Добавить явные примеры значений формата `<PROJECT_ID>` и `<SERVICE_URL>`.
- Убедиться, что инструкции безопасны для публичной публикации.

#### Файл: `docs/runbooks/ops.md`
- Санитизировать примеры команд и переменных окружения (без внутренних идентификаторов).
- Проверить, что инструкции не раскрывают private URLs.

## Тест-кейсы
- Автотесты не требуются (документация).

## Критерии приёмки
- [ ] В репозитории есть LICENSE, CONTRIBUTING, SECURITY, CODE_OF_CONDUCT.
- [ ] `.env.example` отражает требуемые переменные из runbooks/кодовой базы.
- [ ] Runbooks не содержат внутренних project IDs/URL и могут быть опубликованы публично.

## Примечания
- Тип лицензии и кодекс поведения согласовать с владельцем репозитория.
