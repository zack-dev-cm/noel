# Задача 1.3: Базовый E2E (local) с заглушками

## Связь с юзер-кейсами
- UC-01: WebApp onboarding and consent
- UC-02: Observe a live public session

## Цель задачи
Поднять минимальные Playwright тесты (local) для проверки базовой загрузки UI и отображения заглушечных потоков.

## Описание изменений

### Новые файлы
- `tests/e2e/onboarding.spec.ts` — открытие WebApp и проверка базовых элементов.
- `tests/e2e/stream.spec.ts` — проверка отображения заглушечных событий.

### Изменения в существующих файлах
- `playwright.config.ts` — включить headed режим, скриншоты, trace on failure.

### Интеграция компонентов
- Playwright использует `VITE_API_BASE_URL` и локальные порты.

## Тест-кейсы

### End-to-end тесты
1. **TC-E2E-03:** Onboarding экран
   - Ожидаемый результат: отображается заголовок, кнопка consent (заглушка).
2. **TC-E2E-04:** Стриминг заглушек
   - Ожидаемый результат: в ленте есть минимум 2 сообщения.

### Модульные тесты
- Нет (фокус на e2e).

### Регрессионные тесты
- `npx playwright test --headed`

## Критерии приёмки
- [ ] Playwright тесты запускаются локально.
- [ ] Созданы скриншоты для базового потока.

## Примечания
- На этом этапе ожидается статическая заглушка, без реальных API.
