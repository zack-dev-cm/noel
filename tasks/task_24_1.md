# Задача 24.1: Logs readability + latest/current visibility

## Связь с юзер-кейсами
- UC-07: Session replay and resume

## Цель задачи
Сделать логи более читаемыми: показывать самый свежий ход первым, отображать текущий ход даже без ответа, и позволять восстановить полную историю после очистки.

## Описание изменений

### Новые файлы
- Нет.

### Изменения в существующих файлах
- `apps/web/src/App.tsx` — включить текущий незавершённый ход, сортировать по последней активности, корректно очищать историю, добавить восстановление истории.
- `apps/web/src/i18n.ts` — уточнить подписи ролей и добавить бейджи "Live/Latest" + кнопку "Show all".
- `apps/web/src/index.css` — усилить визуальное различие ролей и стили для бейджей.

## Детали по реализации
- Упорядочивать ходы по последнему `seq` в паре (вопрос/ответ).
- Показывать только самый свежий незавершённый ход; остальные неполные не отображать.
- Очистка истории должна использовать последний завершённый `seq`, чтобы не терять текущий ход.
- Кнопка "Show all" восстанавливает просмотр полной истории.

## Тест-кейсы

### End-to-end
1. **TC-E2E-24.1-01:** Самый свежий ход отображается первым, текущий ход виден с бейджем Live.
2. **TC-E2E-24.1-02:** Очистка истории не скрывает текущий ход; кнопка Show all возвращает полный лог.

## Критерии приёмки
- [ ] Логи читаемы: роли различимы, свежий ход сверху.
- [ ] Текущий ход виден даже без ответа.
- [ ] Полная история доступна после восстановления.
